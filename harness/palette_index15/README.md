# P0/index15 palette canonicalization proof

This harness verifies the representation used by the fixed-colour DEBUG HUD.
For each palette segment, the encoder moves a globally brightest existing
usable CRAM colour to palette line 0, index 15 without changing the picture:

1. swap the brightest colour's complete source row with P0 and remap tile
   palette attributes;
2. swap its slot with index 15 inside the new P0 and remap pixel indices;
3. leave transparent index 0 fixed at zero in all four rows.

The player can therefore upload the HUD font once as P0/index15. The top-row
HUD entries are written into the inactive alternating video name table, and
palette switches need no font scan, rewrite, DMA, or VBlank wait.

## Run the focused proof

```sh
tools/python.sh harness/palette_index15/verify.py
```

The synthetic proof deliberately puts the global maximum in non-P0 rows and
different indices. It verifies byte-identical RGB333 output, the complete
60-colour multiset, the no-op tie case, transparent index 0, and the player's
one-time `dbgfont.bin` index1-to-index15 expansion.

To check a newly encoded decision log and its physical PALTAB too:

```sh
tools/python.sh harness/palette_index15/verify.py \
  --dec-log videos/your_stem/decisions.pkl \
  --header out/movieplay/HEADER.DAT
```

This additionally requires every decision-log segment and packed PALTAB entry
to satisfy the global-brightest invariant, checks that all four CRAM index-0
words remain zero, compares PALTAB byte-for-byte with the decision log, and
checks the header's frame-0 palette copy. A pre-canonicalization decision log is
expected to fail and must be regenerated by the current encoder; the packer
also rejects such stale logs.
